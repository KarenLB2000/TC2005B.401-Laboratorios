Preguntas a responder

¿Cuál es la diferencia entre Internet y la World Wide Web?
Internet es un sistema de red conectado globalmente que facilita la comunicación mundial y el acceso a recursos de datos a través de una enorme colección de redes personales, públicas, comerciales, académicas y gubernamentales; World Wide Web es un servicio de recuperación de información de la web (la red informática mundial)(GeeksforGeeks, 2020). Dada la definición de ambos conceptos, se entiende entonces que el Internet sirve como la infraestructura para la Web, siendo este útlimo un servicio sobre trabaja o funciona sobre esa infraestructura.

¿Cuáles son las partes de un URL?
Las partes de una URL son (Aviani, 2019): protocolo HTTP, dominio, ruta y parámetros.

¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE?
De acuerdo con el árticulo de Aviani (2019), el propósito de cada método es el siguiente: GET tiene como propósito la solicitud de los datos de un recurso específico; HEAD, pide una respuesta idéntica a la de una petición GET, pero sin el cuerpo de la respuesta; POST se utiliza para enviar datos a un servidor para crear un recurso; PUT, sirve para actualizar el recurso existente en un servidor utilizando el contenido en el cuerpo de la solicitud; PATCH, es utilizado para aplicar modificaciones parciales a un recurso; y, finalmente, DELETE, borra un recurso en específico.

¿Qué método HTTP se debe utilizar al enviar un formulario HTML, por ejemplo cuando ingresas tu usuario y contraseña en algún sitio? ¿Por qué?
El método que se debe utilizar es POST en virtud de que con éste enviamos los datos del login al servidor.

¿Qué método HTTP se utiliza cuando a través de un navegador web se accede a una página a través de un URL?
El método que se utiliza es el GET, ya que a través de este método, realizamos peticiones al servidor; solicitando un recurso específico (a través de la URL) al servidor web.

Un servidor web devuelve una respuesta HTTP con código 200. ¿Qué significa esto? ¿Ocurrió algún error?
Al leer la descripción porporcionada por MDN Web Docs (2022) sobre el códido de respuesta 200, sabemos que no se trata de un error, puesto que este código hace referencia a que la solicitud HTTP específica ha tenido éxito (se ha completado satisfactoriamente).

¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 404? ¿Por qué?
Depende del caso o situación puede ser responsabilidad del desarollador o bien del usuario. Esto se debe a que el código 404 indica que el servidor no puede encontrar el recurso solicitado (MDN Web Docs, 2022), lo que puede llegar a crear situaciones en donde el usuario trate de acceder algún recurso inexistente o dado un descuido del desarrollador, se de el caso de vínculos rotos en su misma página.

¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 500? ¿Por qué?
En este caso, es responsabilidad del desarrollador, pues, después de todo, dicho código indica que el servidor encontró una condición inesperada que le impide completar la petición (MDN Web Docs, 2022); a lo que el desarrollador debe responder para que el software funcione perfectamente en éste (servidor).

¿Qué significa que un atributo HTML5 esté depreciado o desaprobado (deprecated)? Menciona algunos elementos de HTML 4 que en HTML5 estén desaprobados.
De acuerdo con la definición que proporciona W3C (2018), un elemento o atributo depreciado es aquel que ha quedado obsoleto por construcciones más nuevas. De forma que, en este caso, un atributo HTML5 depreciado hace referencia a un atributo cuyo uso ya no es recomendado para el estándar del HTML5. Algunos ejemplos son: base, font, big, frame, applet, dir, etc.

¿Cuáles son las diferencias principales entre HTML 4 y HTML5?
Las principales diferencias entre ambas versiones sería que en HTML5 se de la introducción de nuevos elementos (em, strong, article, section, ...) y de una nueva sintaxis alternativa (), entre otras cosas.

¿Qué componentes de estructura y estilo tiene una tabla?
En tanto a la estructura de tablas (table), mclibre.org (2019) menciona que una tabla en HTML es un conjunto de celdas (td o th) organizadas en filas (tr) que a su vez se organizan en grupos de filas (thead, tbody o tfoot); ésta puede tener una leyenda (caption) y hacer referencia a las columnas (col y colspan). Por el otro lado, en tanto al estilo de una tabla, ésta puede ser estilizada por medio de un CSS.

¿Cuáles son los principales controles de una forma HTML5?
Entre los controles de formulario originales podemos encontrar a: form, fieldset, legend, textarea, label, button y input. A lo que cabe agregar que los elementos y atributos para formularios en HTML5 proveen un mayor grado de marcado semántico que en HTML4, según lo propone MDN Web Docs (2022) en su artículo de Formularios en HTML5.

¿Cuál es el ciclo de vida de los sistemas de información?
En palabras de Sy Corvo (2020), el ciclo de vida de un sistema de información es un modelo estructurado de gestión de proyectos que describe las fases necesarias para construir un sistema de información, desde el inicio hasta el resultado final.

¿Cuál es el ciclo de desarrollo de sistemas de información?
Siguiendo el hilo de ideas de la pregunta anterior, el Ciclo de Vida del Desarrollo de Sistemas (SDLC) es un método que facilita el desarrollo de los sistemas de información (Viewnext, 2018). Este método comprende una serie de siete etapas o fases, las cuales son: 1) Planificaicón, 2) Análisis de requerimientos, 3) Diseño, 4) Desarrollo, 5) Prueba, 6) Integración y ejecución, 7) Operación y mantenimiento.
